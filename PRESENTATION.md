## Слайд 5: Ключевая фича — шифрование

### AES-256-GCM

**Industry standard** для аутентифицированного шифрования.

```python
class CryptoService:
    def encrypt(self, data):
        iv = os.urandom(12)  # Случайный IV
        encrypted = aesgcm.encrypt(iv, data, None)
        # Auth tag добавляется автоматически
        return {'encrypted': ..., 'iv': ..., 'auth_tag': ...}
```

### Что это даёт?

| Свойство | Описание |
|----------|----------|
| **Конфиденциальность** | Никто не прочитает файл без ключа |
| **Аутентичность** | Изменения данных обнаруживаются |
| **Скорость** | Быстрее, чем AES-CBC + HMAC |

## Слайд 6: Хранение в Discord

### Почему Discord?

- ✅ **Бесплатно** — нет лимитов на хранение
- ✅ **Постоянно** — файлы хранятся бессрочно
- ✅ **Стабильно** — Discord CDN надёжен
- ✅ **Быстро** — глобальная CDN сеть

### Как это работает?

```
Файл → Шифруется AES-256 → Делится на чанки
                                ↓
                          Discord Webhook
                                ↓
                          Discord CDN URL
                                ↓
                          Сохраняется в БД
```

### Особенности
- Лимит 25 МБ на файл → автоматическое разбиение на чанки
- Retry с экспоненциальной задержкой при ошибках
- Поддержка rate limiting (429 + Retry-After)

## Слайд 8: ORM модели

```python
class User(UserMixin, db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(80), unique=True)
    password_hash = db.Column(db.String(256))
    role = db.Column(db.String(20), default='user')
    quota_bytes = db.Column(db.BigInteger, default=0)
    
    files = db.relationship('File', backref='owner')
    folders = db.relationship('Folder', backref='owner')


class File(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(255))
    size = db.Column(db.BigInteger)
    parts_json = db.Column(db.Text)  # JSON со ссылками
    iv = db.Column(db.String(32))
    trashed_at = db.Column(db.DateTime)  # Корзина
```

### Связи
- `User` 1:N `File`, `Folder`, `Share`
- `Folder` 1:N `File`, self-ref для вложенности

## Слайд 10: Безопасность

### 🔐 Защита данных
- AES-256-GCM для файлов
- bcrypt для паролей (через Werkzeug)
- HTTPS-ready (через Nginx в production)

### 🛡️ Защита API
- `@login_required` на всех приватных эндпоинтах
- `@admin_required` для админки
- Проверка владения ресурсом
- Валидация всех входных данных

### 🚫 Защита от атак
- CSRF через session cookies
- SQL Injection невозможна (ORM с параметризованными запросами)
- XSS — escape HTML в frontend
- Sanitize имён файлов
