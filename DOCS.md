## Cloud Storage — облачное хранилище с шифрованием

---

## 1. Введение

### 1.1 Идея проекта

Cloud Storage — это веб-приложение для безопасного облачного хранения файлов. Главная особенность проекта в том, что **все файлы шифруются на стороне сервера** перед отправкой во внешнее хранилище (Discord CDN), что обеспечивает полную конфиденциальность данных пользователей.

### 1.2 Решаемые задачи

- **Бесплатное облачное хранилище** — использование Discord CDN вместо платных сервисов вроде AWS S3
- **Шифрование данных** — никто, кроме владельца ключа, не может прочитать файлы
- **Простой веб-интерфейс** — современный UI с поддержкой drag-and-drop
- **Многопользовательский режим** — система авторизации, квоты, роли
- **Расшаривание** — публичные ссылки с паролем и сроком действия

### 1.3 Целевая аудитория

- Пользователи, которым нужно бесплатное и приватное облачное хранилище
- Команды, которым нужно делиться файлами с защитой паролем
- Студенты и преподаватели для обмена учебными материалами

## 5. Описание ключевых модулей

### 5.1 Шифрование (services/crypto.py)

Используется **AES-256-GCM** — современный алгоритм аутентифицированного шифрования.

**Принцип работы:**
1. Из мастер-ключа извлекается 256-битный ключ через SHA-256
2. Для каждого шифрования генерируется случайный 96-битный IV (nonce)
3. Данные шифруются с помощью `AESGCM.encrypt()`
4. GCM автоматически добавляет 128-битный auth tag для проверки целостности

**Преимущества AES-GCM:**
- Конфиденциальность данных
- Аутентификация (любые изменения зашифрованного текста обнаруживаются)
- Высокая производительность

```python
# Пример использования
crypto = CryptoService(master_key="secret-key")
result = crypto.encrypt(b"My file data")
# result = {'encrypted': bytes, 'iv': str, 'auth_tag': str}

decrypted = crypto.decrypt(result['encrypted'], result['iv'], result['auth_tag'])
```

### 5.2 Хранение в Discord (services/discord.py)

Discord предоставляет бесплатное хранилище через webhook'и (до 25 МБ на файл, бессрочное хранение на CDN).

**Алгоритм загрузки:**
1. Файл шифруется
2. Если файл >25 МБ — разбивается на чанки
3. Каждый чанк загружается отдельным запросом к webhook
4. Информация о частях (URL, IDs, IV, auth tag) сохраняется в БД

**Обработка ошибок:**
- Retry с экспоненциальной задержкой при сетевых ошибках
- Поддержка rate limiting (HTTP 429) с Retry-After
- Серверные ошибки 5xx тоже повторяются

### 5.3 ORM модели

Все модели наследуются от `db.Model` (SQLAlchemy):

**User** (модель пользователя):
- Хранит хэш пароля (bcrypt через Werkzeug)
- Поддерживает роли: `user` / `admin`
- Поле квоты для ограничения хранилища
- Поле `active` для блокировки

**File** (модель файла):
- Хранит метаданные и зашифрованные части (JSON)
- Поддерживает soft delete через `trashed_at` (корзина)
- Окончательное удаление через `deleted_at`

**Folder** (модель папки):
- Self-referential relationship для иерархии
- Уникальность имени в пределах родительской папки

**Share** (публичная ссылка):
- Криптографически стойкий токен (`secrets.token_urlsafe`)
- Опциональный пароль (хэшируется отдельно)
- Срок действия и лимит скачиваний

### 5.4 REST API

Все основные операции доступны через REST API:

**Файлы:** `/api/files`, `/api/files/<id>`, `/api/files/<id>/upload`, `/api/files/<id>/download`, `/api/files/<id>/preview`

**Папки:** `/api/folders`

**Корзина:** `/api/trash`, `/api/files/<id>/restore`

**Поиск:** `/api/search?q=...&category=...`

**Публичные ссылки:** `/api/files/<id>/share`, `/s/<token>`

**Админ:** `/admin/api/users`, `/admin/api/stats`

## 6. Безопасность

### 6.1 Защита данных

- **AES-256-GCM** для всех файлов в хранилище
- **bcrypt** для паролей пользователей (через Werkzeug)
- **CSRF защита** через Flask-Login session cookies
- **Валидация всех входных данных** на сервере (services/validators.py)

### 6.2 Защита API

- Все API защищены `@login_required`
- Админские эндпоинты требуют `@admin_required`
- Проверка владения ресурсом (нельзя получить чужой файл по ID)

### 6.3 Защита от загрузки опасных файлов

- Sanitize имён файлов (убираются `<>:"/\\|?*`)
- Лимит размера файла (`MAX_FILE_SIZE_MB`)
- Проверка квоты пользователя
